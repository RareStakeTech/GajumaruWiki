## Documentation Principles

1. **JUST WRITE** — "imperfect content is infinitely better than nonexistent content"
2. **PICTURES** — Articles should include visual diagrams
3. **QUICKREF** — Practical "getting started" sections with examples
4. **DEMOS DEMOS DEMOS** — Functional code examples throughout
5. **CENTRALIZE** — All technical docs consolidated on the wiki
6. **LINK** — Provide source code references and receipts
7. **VIDEO** — Conceptually complex topics include video explanations

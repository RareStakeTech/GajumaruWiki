# RIPA Model

RIPA stands for **Resource, Infrastructure, Platform, Application** — the four-layer deployment model for the Gajumaru ecosystem.

## Layers

| Layer | Gajumaru Component | Description |
|-------|-------------------|-------------|
| **Resource** | [[groot]] | Base consensus layer; energy-efficient block production |
| **Infrastructure** | [[associate-chains]] | Sovereign/independent chains with native interoperability |
| **Platform** | Smart contract environment | Sophia-language contracts; permissionless post-MainNet |
| **Application** | [[gajumaru-product-suite]] | End-user products (GajuDEX, GajuMarket, GajuPay, etc.) |

## Status
Fully operational RIPA model delivered at MainNet launch (April 6/7 2026). Prior to MainNet, only R and partial I layers were live.

## Confidence Note
The RIPA framing is used in QPQ communications but detailed technical specifications of layer boundaries are not yet in the wiki. Confidence rated medium pending Un-White Paper ingest.
